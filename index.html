Getting Started

What is HTML?

"HTML" or "Hyper-Text-Markup-Languange" is the language that devolopers use to create web pages like this one. HTML is a series of elements that provide content to a page including text and images. While there are ways to add minor styling to HTML content, at its core, HTML is comprised of all the "raw" information that the creater of a page would like to convey. Before we get started lets take a look at several definitions and pieces of information that are important to understand before diving deeper in to the world of HTML:

  • Syntax- The important order in which somenting is created and used.
  • Nesting- Placing an item or group of items within another item.

HTML Syntax: Parent and Child Elements

It is helpful to think of "HTML" as a human body, where a body would be a main "parent-element" which contains a grouping of secondary "child-elements" like arms and legs, this is useful to a developer so they can keep their HTML document well organized. This technique is also helpful later on when a developer wants to style the content of their page, now they can choose to either alter a whole group of elements (using parent-element) or just one specific element (child-element). If we were to create a single part of the human body using HTML syntax, it might look something like this:

<human>

<body>

<arms>

<right-arm>

<hand>

<fingers>

<thumb></thumb>

<index></index>

<middle></middle>

<ring></ring>

<pinky></pinky>

</fingers>

</hand>

</right-arm>

</arms>

</body>

</human>

In the example above you can see that each element has an opening tag and a coorisponding closing tag that contain a group of other elements this is a great example of a parent-child relationship.

To recap everything so far:

  • Each element that is enclosed in the tags of another element will become the child-element with the enclosing element being the parent-element i.e. "body" is the child-element of "human", its parent-element.
  • Parent elements can contain more than one child element, this can be seen where "thumb", "index", "middle", "ring", and "pinky" all being the child-elements nested within the parent-element "fingers" or if we were to add a "left-arm" element we would make sure that it would be nested after "arms", alongside "right-arm", and before any other element of its own.
  • It is important to note that, theoreticlly, the process of nesting parent and child elements could continue indefinitly with the developer creating and nesting as many elements as they choose. While this might be the case, the best developers only nest as much as they need to as this will help keep their documents well organized.

Setting Up HTML Documents

In the previous example we used the parts of the human body as placeholders for true HTML elements, this is a great example as it helps us visualize the relationship between parent and child elements with something that we are all familiar with, this gives us a better idea of the overall setup and flow of an HTML document. Now that we understand this important relationship we can take a look at an example of a very small clip of actual code that would be used as the primary framework to an HTML document:

<!DOCTYPE html>

<html>

<head>

<title>

Title of the Page Goes Here

</title>

</head>


<body>

Content of the Body Goes Here; Headers, Paragraphs, and Images.

</body>

</html>

When evaluating the code above it is important to focus on each element as they all play very specific roles that are required to set up our web page.

Lets take a look at each individual element:

  • !DOCTYPE html- !DOCTYPE is very important and is used at the top of every HTML document, it is what tells the browser or other code reader that we have written our code using HTML and would like it to read and display the code in that format.
  • html- this is our HTML document, the document itself contains two main child elements:
    • head- the head section of an HTML document contains information about the page that is not visable on the primary page itself, this includes the elements such as the title of the page that will appear in the tab of a browser.
    • body- the body section of an HTML document is where a developer will spend most of their time developing the "raw" data that they want a viewer to see on the webpage itself, this will include headers, paragraphs, lists, and images, all of which we will learn about down below.

Now that we know how to set up our HTML document lets learn how to start adding some content to our page!

Adding Content

Reading Between the Tags

In the previous section we learned the basic structure and syntax of HTML, this is useful when we are setting up our page, however, every webpage needs content. Webpages consist of two primary types of content; text and images. In this section we will learn how to add several of the basic forms of text to our webpage as well as how to add an image.

Headers

Headers can help us organize the layout and readability of our webpage by creating titles that provides information of the respective section. There are six main headers, each of which is smaller than the next starting with h1 being the largest down to h6 being the smallest. To set up a header section we will will start with an opening set of header tags with h1-6: "<h1>" followed by content: "<h1>Content" and finally we will close off the header with closing tags: "<h1>Content</h1>". Below you will see an example of how to add each of the headers to your HTML file.

<h1>Header 1</h1>

<h2>Header 2</h2>

<h3>Header 3</h3>

<h4>Header 4</h4>

<h5>Header 5</h5>

<h6>Header 6</h6>

Headers each have a default size which can be changed later on, but it is important to note that we use headers just like we set up an html document, where smaller headers of the same size will be nested in a larger header parent element, this helps keep our document sections organized and readable. Lets not get too caught up on this concept for now as it will be seen in all of the following sections.

Paragraphs

Now that we know how to add headers to set up the sections of our web page, lets take a look on how to add the information in those sections. To add text we use 'p' or 'paragraph tags' just like headers we will use an opening and closing tag that is then filled with information inbetween to create a paragraph section like this one. Take a look below:

<h1>Header 1</h1>

<h2>Header 2</h2>

<p>This is paragraph text!</p>

<h2>Header 4</h2>

Like we were talking about earlier it is important that headers and text are used in a successive parent-child relationship... if we wanted each 'h2' to have its own sub headers/sections it would be confusing if we used 'h6' for the next set of headers because we liked the size, instead, if we wanted to create new sub headers/sections within the 'h2' section we would use proper syntax and use 'h3' changing the size later if we would like.

Lists

Lists can be a very helpful way to display information as they often provide more concise clips of information. We can use two primary lists ordered or unordered depending on what type of information we are trying to present in our list. we use unordered lists 'ul' to present a list that doesent need to have any specific order or count to the items it contains. Ordered lists 'ol' are used when rank, order, or count of items in the list is needed. Each list is set up with opening and closing tags that contain list items 'li', one for each item we would like on our list. Take a look below to see how we set up a list:

<h1>Lists</h1>

<h2>Unordered List</h2>

<ul>

<li>List Item</li>

<li>List Item</li>

<li>List Item</li>

</ul>


<h2>Ordered List</h2>

Images

Now that we can add text to our page lets also add an image as well. To add an image we only use one opening tag <img> that will contain two main elements for now:

  • src = " " - this is where we will add our url to link the image we would like to add to the page into the page. When linking an image from the internet it is important that the whole url makes it into the quotations
  • alt = " " - if something is wrong with our link an alternate text description will be needed to tell the viewer what was going to be there. This is more importantly used for accessability for those who require the use of a screen reader, for example, one who is visually impared.
  • All together the tag will look something like this: <img src="https://www.cute-images.com/cat" alt="cute cat photo">
